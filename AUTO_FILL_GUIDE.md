# 🤖 后台自动补充功能使用指南

## 功能概述

系统会在检测到空闲状态时，自动在后台补充书籍信息，并在检测到用户操作时暂停，实现智能的后台处理。

## 🎯 主要特性

### 1. 智能空闲检测
- 自动跟踪用户活动（页面访问、API调用等）
- 可配置的空闲时间阈值（默认5分钟）
- 自动识别用户操作并暂停任务

### 2. 自动任务管理
- 空闲时自动启动补充任务
- 检测到活动时自动暂停
- 任务完成后自动开始下一批

### 3. 灵活配置
- 启用/禁用自动补充
- 自定义空闲阈值
- 自定义每批处理数量
- 自定义检查间隔

## 📋 使用步骤

### 1. 开启自动补充

1. 以**管理员**身份登录
2. 点击顶部的 **⚙️ 自动补充** 按钮
3. 勾选 **启用后台自动补充**
4. 根据需要调整配置参数
5. 配置会自动保存

### 2. 配置参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **空闲时间阈值** | 300秒 | 系统无操作多久后视为空闲 |
| **每批处理数量** | 10本 | 每次自动处理的书籍数量 |
| **检查间隔** | 60秒 | 每隔多久检查一次是否需要启动任务 |

### 3. 工作流程

```
用户活动 → 更新活动时间
    ↓
后台线程每60秒检查一次
    ↓
计算空闲时间
    ↓
空闲时间 < 300秒 → 暂停任务（如果正在运行）
空闲时间 ≥ 300秒 → 启动/恢复任务
    ↓
随机选择10本缺失信息的书籍
    ↓
调用AI逐个补充
    ↓
更新本地和Calibre数据库
    ↓
任务完成 → 等待下次检查
```

## 🔍 状态监控

配置页面会实时显示：
- **当前状态**：未启用/等待启动/运行中/已暂停
- **最后活动**：最近一次用户操作的时间

状态颜色：
- 🔴 灰色：未启用
- 🟢 绿色：运行中
- 🟠 橙色：已暂停
- 🔵 蓝色：等待启动

## ⚙️ 高级配置建议

### 根据使用场景调整

```

批处理量：20本
检查间隔：30秒
```
适合：经常离开电脑短暂休息的场景

**2. 办公使用（多用户）**
```

批处理量：10本
检查间隔：120秒
```
适合：白天使用频繁，希望在午休或下班后处理

**3. 服务器部署（24小时运行）**
```
空闲阈值：300秒（5分钟）
批处理量：15本
检查间隔：60秒
```
适合：持续运行，快速响应空闲状态

## 🛡️ 安全特性

1. **只跟踪有意义的活动**
   - 排除静态文件请求
   - 排除健康检查请求
   - 排除自动补充相关的API

2. **智能任务管理**
   - 检测到活动立即暂停
   - 任务异常自动恢复
   - 避免重复处理

3. **资源控制**
   - 每次只处理少量书籍
   - 可随时禁用功能
   - 后台线程守护模式

## 🔧 故障排查

### 问题1：自动补充没有启动
**检查清单**：
- [ ] 是否勾选了"启用后台自动补充"
- [ ] 是否有足够的空闲时间
- [ ] 是否有需要补充的书籍
- [ ] 查看应用日志确认后台线程已启动

### 问题2：任务频繁暂停
**可能原因**：
- 空闲阈值设置过短
- 其他服务调用API
- 浏览器自动刷新

**解决方案**：
- 增加空闲时间阈值到600秒或更长
- 检查是否有自动化脚本在运行

### 问题3：处理速度慢
**优化建议**：
- 增加每批处理数量（最多50本）
- 减少检查间隔（但不要低于30秒）
- 检查AI服务响应速度


2. **检查处理进度**
- 批量AI补充页面可以看到自动任务的处理日志
- 标记为"auto: true"的任务是自动任务

3. **数据库统计**
```python
# 查询缺失信息的书籍数量
from app import app, db, Book
with app.app_context():
    count = Book.query.filter(db.or_(
        Book.author == None,
        Book.description == None
    )).count()
    print(f"还有 {count} 本书需要补充信息")
```
